执行上下文与作用域

提示

javascript在执行语句前，金果榄一系列的“准备”，为代码执行创造一个“教室” -- 执行上下文

this value 文本环境-- 点名册

# 执行栈

  • 执行栈栈顶的执行上下文称为当前执行上下文
  • JS 代码总是在当前上下文中运行
  • 意思是 JS 代码中需要用到的资源，到当前执行上下文上查找

4 种情况会创建新的执行上下文（ES6）

  • 进入全局代码
  • 进入 function 函数体代码
  • 进入 eval 函数参数指定的代码
  • 进入 module 代码
    console.log(foo); //undefined
    if(false){
        var foo = "foo";
    }

# step1. 构建全局执行上下文

全局执行上下文 由两部分组成

  1. 全局对象,即 window对象
  2. 全局 scope

提示

  1. var 和 function 声明创建在 全局对象 中
  2. let , const , class 声明的变量创建在 全局scope 中
  3. 找变量的方式：先从 全局scope 中找变量，如果找不到再到 全局对象 中查找
  4. 创建函数对象，函数对象体内 会保存该函数创建时 的执行上下文的文本环境
  5. 函数的执行上下文 只有 scope
    let Function = "let 定义覆盖 function";
    console.log(Function);//let 定义覆盖 function
    console.log(window.Function);//f函数

# step2. 找到所有声明

  1. 找到所有的 非函数 中的 var 声明
  2. 找到所有的顶级函数声明（即不包括在大括号内的所有函数声明）不是函数表达式
  3. 找到顶级 let const class 声明

# step3. 变量名字重复处理

  1. let const class 声明的名字之间不能重复
  2. let const class 和 var function的名字不能重复
  3. var 和 function 名字重复的，function 声明的函数名优先

# step4. 创建绑定

  • 登记并初始化var为 undefined
  • 顶级函数声明：登记 function 名字，并初始化为新创建函数对象
  • 块级中函数声明：登记 function 名字，初始化为 undefined
  • 登记let const class ，但未初始化 （所以无变量提升）

# 作用域

作用域是解析变量名的一个集合，就是当前执行上下文（也可以是当前上下文的词法环境）

  • 全局作用域就是全局运行上下文
  • 函数作用域就是函数运行上下文

函数调用时的执行上下文看“身世” -- 函数在哪里创建，就保存哪里的运行上下文

函数的作用域是在函数创建的时候决定的而不是调用的时候决定

提示

并非根据调用嵌套形成作用域链，而是根据`函数创建嵌套`形成作用域链，也就是函数的书写位置形成作用域链，因此称为 *词法作用域*
```js
    function foo(){
        console.log(a);//2
    }
    function bar(){
        var a = 3;
        foo();
    }
    var a = 2;
    bar();
```

# 块级作用域 { }

# step1.

创建新的词法环境，链接在原来记录的词法环境之前

# step2. 找到声明

  • 找到所有的顶级函数声明
  • 找到 let const 声明

# step3. 名字重复处理

  • function 和 let const 名字不重复
  • let const 直接的名字不能重复

# step4. 创建绑定

  • 登记 function 名字，并初始化为新创建函数对象
  • 登记let const 但未初始化

# step5. 执行语句

提示

当执行完 `{ }` 内的语句后，会 立即销毁 之前创建的词法环境
如果 `{ }` 内有创建函数，在代码块执行完退出时，会将内部创建的函数名与`全局对象`中比较，如果全局对象中有同名的变量则将该函数的值赋给全局对象中得同名变量，如果在全局对象中没有找到对应的变量则不进行赋值。


    //1.
    console.log(foo);//undefined 
    if(false){
        //代码块中的对象也是有提升,在构建全局执行上下文的时候提升
        function foo(){
            console.log('123');
        }
    }

    //2.
    console.log(foo);//error not defined
    if(false){
        // function foo(){
        //     console.log('123');
        // }
    }

    //3.
    console.log(foo);//undefined
    if(false){
        function foo(){
            console.log('123');
        }
    }
    foo();//error

    //4.
    let foo;
    if(true){
        function foo(){
            console.log('123');
        }
    }
    foo();//error

    //5.
    var foo;
    if(true){
        function foo(){
            console.log('123');
        }
    }
    foo();//123

    //6.
    console.log(foo);
    if(true){
        function foo(){
            console.log('123');
        }
    }
    foo();//123
